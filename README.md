### Avvertenze Legali

- L'accesso non autorizzato a sistemi informatici è un **reato penale** in Italia (art. 615-ter c.p.) e nella maggior parte delle giurisdizioni

- L'installazione di dispositivi per intercettare comunicazioni è vietata dalla legge
- La diffusione di malware è un reato (art. 615-quinquies c.p.)

## Scopo del Progetto

Questo materiale è stato creato per un'attività di autogestione scolastica con l'obiettivo di:
- Educare studenti sui rischi della sicurezza informatica
- Dimostrare tecniche di attacco comuni in modo controllato
- Insegnare buone pratiche di protezione
- Aumentare la consapevolezza sui pericoli digitali

## Ambienti Controllati

Tutte le dimostrazioni devono essere eseguite **ESCLUSIVAMENTE** in:
- Macchine virtuali isolate
- Reti di test disconnesse da Internet
- Laboratori controllati con autorizzazione
- Ambienti sandbox dedicati

**NON testare MAI su:**
- Reti pubbliche o condivise
- Sistemi di produzione
- Dispositivi non di proprietà
- Reti scolastiche/aziendali senza autorizzazione scritta

## Contenuti

Il progetto copre le seguenti aree:

1. **Phishing e Ingegneria Sociale**
   - Demo di attacchi phishing
   - Tecniche di riconoscimento
   
2. **Password Security**
   - Cracking e sicurezza password
   - Best practices

3. **Network Security**
   - Packet sniffing
   - Rischi WiFi pubblico

4. **Physical Security**
   - Keylogger
   - Rubber Ducky

5. **Malware Awareness**
   - Trojan e Ransomware
   - Prevenzione
